# Al Qaysar Menu System - How It Works

## Overview

Your system consists of **two main components**:

1. **Website (menu.html)** - Customer-facing QR menu
2. **Mobile App (staff.html)** - Staff menu editor

Both are deployed to Vercel and share the same Firebase database.

---

## 🌐 Website: Customer QR Menu (menu.html)

### Purpose
Customers scan a QR code that takes them to your Vercel website showing the current menu.

### How It Works
1. Customer scans QR code → Opens `menu.html` on Vercel
2. Page loads menu data from Firebase Realtime Database
3. Displays only dishes that staff have marked as "available"
4. Shows dishes organized by sections (Appetizers, Grills, Chinese, etc.)
5. Updates in real-time when staff changes the menu

### Key Features
- Dark theme design with gold accents
- Arabic and English dish names
- Prices in Saudi Riyals (ر.س)
- Tabbed navigation by food category
- Shows "Last updated" timestamp
- Responsive mobile-first design

### Firebase Connection
```javascript
// Reads from: firebase.database().ref('menu')
// Structure:
{
  "menu": {
    "dishes": { /* all available dishes */ },
    "updatedAt": "2026-04-20T10:30:00.000Z"
  }
}
```

## 📱 Mobile App: Staff Menu Editor (staff.html)

### Purpose
Staff use this Android app to select which dishes are available today.

### How It Works
1. Staff opens the app on their Android device
2. Enters PIN (default: "1234") to login
3. Sees full list of all dishes organized by section
4. Toggles dishes on/off with checkboxes
5. Clicks "Update guest menu" to save changes
6. Changes appear instantly on customer menu (menu.html)

### Key Features
- PIN-protected access (staff only)
- Search dishes by name (English or Arabic)
- "All on" / "All off" buttons per section
- Shows count of selected dishes
- Light theme for easy reading
- Works offline (PWA with service worker)

### Firebase Connection
```javascript
// Writes to: firebase.database().ref('menu')
// Saves both:
// 1. Available dish IDs
// 2. Full dish data (name, price, section)
```

## 📦 Android App Structure

The staff app is packaged as an Android APK using **Capacitor**.

### File Structure
```
staff.html          → Original staff editor (web version)
www/
  └── index.html    → Copy of staff.html (used by Android app)
  └── manifest.json → PWA manifest
  └── service-worker.js → Offline caching
android/            → Native Android wrapper (Capacitor)
```

### How Android App Works
1. Capacitor wraps `www/index.html` in a native Android WebView
2. App runs the web page as if it's a native app
3. Has access to device features (camera, storage, etc.)
4. Can be installed from APK file

## 🚀 Deployment

### Vercel (Website)
- **URL**: Your Vercel domain
- **Routes**:
  - `/` or `/menu` → menu.html (customer menu)
  - `/staff` → staff.html (staff editor web version)

### Android App
- Built using Capacitor
- APK file installed on staff tablets/phones
- Loads `www/index.html` (which is a copy of staff.html)

## 🔄 Workflow: How Staff Updates Menu

1. **Morning**: Staff opens Android app
2. **Login**: Enters PIN (1234)
3. **Select**: Toggles on dishes available today
4. **Save**: Clicks "Update guest menu"
5. **Firebase**: Saves changes to database
6. **Instant**: Customer menu updates in real-time

## 🔐 Security Notes

### Current Setup
- Staff PIN: `1234` (hardcoded in staff.html)
- Firebase: Public read/write access (no authentication)

### Recommendations
1. **Change PIN**: Edit `STAFF_PIN` in staff.html before building APK
2. **Firebase Rules**: Add security rules to restrict write access
3. **HTTPS**: Vercel provides HTTPS by default ✓

## 🛠️ Making Changes

### Update Menu Items
Edit the `MENU` array in `staff.html` (around line 200)

### Change Styling
Edit the `<style>` section in:
- `menu.html` for customer menu
- `staff.html` for staff app

### Update Firebase Config
Replace `firebaseConfig` object in both files

### Rebuild Android App
```bash
# 1. Update staff.html
# 2. Copy to www/
cp staff.html www/index.html

# 3. Sync with Android
npm run sync

# 4. Build APK
npm run build:android
```
